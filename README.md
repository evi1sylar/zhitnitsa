# 🥖 Житница — Сеть пекарен

Django-приложение для управления сетью пекарен с каталогом продукции, заказами и админкой.

## 🚀 Быстрый запуск

### Вариант 1: Автоматический (рекомендуется)

```bash
# Просто запустите файл
start.bat
```

Скрипт автоматически:
- ✅ Проверит установку Python
- ✅ Создаст виртуальное окружение
- ✅ Установит все зависимости
- ✅ Применит миграции БД
- ✅ Запустит сервер на http://localhost:8000

### Вариант 2: Ручной

```bash
# 1. Создать виртуальное окружение
python -m venv venv

# 2. Активировать
venv\Scripts\activate  # Windows
source venv/bin/activate  # Linux/Mac

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Применить миграции
python manage.py migrate

# 5. Запустить сервер
python manage.py runserver 8000
```

- Python 3.10+

## 🔐 Админка

**URL:** http://localhost:8000/admin/

**По умолчанию:**
- Username: `admin`
- Password: `admin123`

**Создать админа:**
```bash
python manage.py createsuperuser
```
Или запустите `create_superuser.bat`

## 📱 Функционал

### Для пользователей:
- Просмотр продукции с фильтрацией по тегам
- Оформление заказов с выбором пекарни
- Оставление отзывов (на модерации)
- Личный кабинет с заказами
- Сброс пароля

### Для администраторов:
- Управление продукцией (CRUD)
- Управление пекарнями с фото
- Модерация отзывов
- Управление пользователями
- Обработка заказов
- Редактирование страницы "О нас"

## 🛡️ Безопасность

- CSRF защита
- Валидация загружаемых файлов (5MB лимит)
- XSS защита через Django templates
- Декораторы `@staff_required` для админских функций
- Security headers и CSP

Полная документация: [SECURITY.md](SECURITY.md)

## 🔄 Обновление

```bash
# Получить последние изменения
git pull origin main

# Обновить зависимости
pip install -r requirements.txt --upgrade

# Применить новые миграции
python manage.py migrate
```

## 🐛 Известные проблемы

- При первом запуске убедитесь, что порт 8000 свободен
- Если есть ошибки миграций — удалите `db.sqlite3` и запустите `migrate` заново
